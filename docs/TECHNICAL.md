# 核心技术实现

> 返回 [README](../README.md)

本文档详述 FOC 关节驱动器的核心技术环节:SVPWM 调制、电流采样、编码器校准对齐、角度中断同步、三环控制、CAN 协议。

---

## 1. SVPWM 共模注入调制

SVPWM(空间矢量脉宽调制)负责把 dq 坐标系下的电压指令,转成三相 PWM 占空比。本项目没有采用经典的"判扇区 + 七段式查表"方案,而是用数学等价的 min-max 共模注入法。

两者输出完全一致,但注入法没有易错的扇区映射表——七段式查表的扇区编号约定稍有出入就会导致某一相焊死,而注入法只需比较和加减,代码更短、更不易错(这一点是踩过坑换来的,见 [DEBUGGING.md](./DEBUGGING.md))。

**共模注入的价值**:通过向三相注入相同的零序(共模)分量,把对地电压波形从正弦削成"马鞍形",腾出母线余量。三相共模相减抵消,不影响相间电压和转矩,但相电压上限从 SPWM 的 `Vdc/2` 提升到 `Vdc/√3`——母线利用率从 78.5% 提到 90.7%,提升约 15.5%。

```c
void SVPWM(float U_alpha, float U_beta)
{
    // 逆 Clark: αβ → 三相参考电压
    float Ua = U_alpha;
    float Ub = -0.5f * U_alpha + SQRT3_2 * U_beta;
    float Uc = -0.5f * U_alpha - SQRT3_2 * U_beta;

    // 注入法核心: min-max 共模(七段式的数学等价实现)
    float Vmax = fmaxf(Ua, fmaxf(Ub, Uc));
    float Vmin = fminf(Ua, fminf(Ub, Uc));
    float Vcom = 0.5f * (Vmax + Vmin);   // 共模分量

    // 减共模 + 归一化到占空比[0,1] + 居中 × ARR
    float duty_a = ((Ua - Vcom) / VBUS + 0.5f) * PWM_ARR;
    float duty_b = ((Ub - Vcom) / VBUS + 0.5f) * PWM_ARR;
    float duty_c = ((Uc - Vcom) / VBUS + 0.5f) * PWM_ARR;

    // 限幅后写入三相 CCR(中心对齐 25kHz)
    // ...
}
```

## 2. 电流采样:谷底同步 + DMA

FOC 电流环要求采到的是真实的相电流,而功率级在 25kHz 高速开关,采样时刻选错就会采到开关噪声。本项目用 TIM3 TRGO 触发 ADC + DMA 循环搬运,把采样时刻锁定在 PWM 谷底。

**为什么是谷底**:中心对齐 PWM 的谷底时刻,三相下桥臂全部导通,电感电流通过下管 + 分流电阻续流(电感电流不能突变,撤压后靠续流维持),此时分流电阻上的压降正比于真实相电流;且谷底远离开关沿,避开了 dV/dt 噪声。这是 inline 采样的最佳时刻。

链路:

```
相线 → 10mΩ 分流电阻 → INA240A2(增益50) → ADC1(IN0/IN4)
     → TIM3 TRGO 谷底触发 → DMA Circular 自动搬运
```

电流换算(INA240A2 增益 50,Rshunt = 10mΩ):

```c
i[A] = (adc_raw - offset) * 3.3f / 4096.0f / (50.0f * 0.01f);
```

`offset` 为每次开机零偏校准值:拉低 EN_GATE(功率级高阻、零电流)→ 采 1000 次取平均。必须在 EN_GATE 拉低时校准,否则零偏被导通电流污染,运行期所有电流计算永久带误差。

**采样正确性自检**:三相电流和 `ia + ib + ic ≈ 0`(基尔霍夫电流定律)——这是验证电流采样对错的黄金标准,通电实测约 `1.28 + 0.04 - 1.50 ≈ 0`。

## 3. 编码器零位校准与相位对齐

编码器只知道转子的机械角,但 FOC 定向需要电角度 θe,且必须知道"编码器读数的零位"对应"电机的哪个电角度"。这中间差一个固定偏移,必须在运行前标定出来,否则磁场指向错误、电机堵转或乱转。

本项目分两步标定:

**① 编码器零位校准(强制定向法)**:上电时强制输出一个指向电角度 0 方向的电压(`Ud` 方向),把转子"吸"到电气零点锁定,此刻读到的编码器机械角就是零位偏移 `theta_offset`。

```c
theta_e = (theta_offset - theta_m) * POLE_PAIRS;  // 机械角 → 电角度(×7对极)
```

校准必须在足够电流下进行(实测限流 ≥1A,转子才吸得到位;0.5A 时被电源 CC 卡住、吸不动、offset 乱跳)。

**② 相位对齐补偿**:仅有 offset 仍不自启动、需手拨。用开环匀速旋转拖动电机,对比"指令电角度"与"编码器算出的电角度",测出一个固定相位偏移 ≈1.3 rad(`theta_comp`)。降低开环电压验证该偏移不随电压变化,确认是实打实的对齐偏移而非负载角。

```c
theta_e = (theta_offset - theta_m) * POLE_PAIRS + theta_comp;
```

> 校准每次上电执行一次;由于有 7 个电气零点,每次吸到的 offset 可能不同,但相差整数个电气周期,×极对数取模后等价,不影响定向。

## 4. 编码器角度的中断同步架构

FOC 的所有坐标变换都依赖电角度 θe,而 θe 的新鲜度和同步性直接决定控制质量。本项目最终方案是:θe 在 25kHz 控制中断内,通过 SPI + DMA 非阻塞读取,与电流采样同一节拍。

早期版本把 SPI 读取放在主循环(异步更新),中断每 20μs 拿到的 θe 新鲜度随主循环负载波动——表现为"加不加 printf 电机行为就变"。根因是快慢环不同步:控制中断用的是一个更新速率不固定的角度,导致相位抖动、转矩方向不稳。

解决思路是把 θe 读取也纳入中断节拍:

```
TIM3 中断(分频后)→ 读上一拍 DMA 结果 → 算 θe → FOC 全套运算 → 踢下一拍 DMA
```

SPI 读取经 DMA 分频到 ~10kHz(避开 HAL 状态机的高频重入边界),并加入错误自愈机制保证读取链路出错可自动恢复。

这一架构改造是开发的核心节点——它把"算法正确但行为飘忽"的系统,变成了确定性可控的实时系统。具体的 HAL+DMA 锁死排查与自愈机制实现,见 [DEBUGGING.md](./DEBUGGING.md)。

## 5. 三环级联与位置环 PD 直控(最终方案)

FOC 的完整伺服控制是三环级联,从内到外:电流环 → 速度环 → 位置环,带宽逐层递减(内环带宽需高于外环 5~10 倍,保证内环对外环"足够快")。本项目三环均实现并验证过,最终位置环采用 PD 直控扭矩结构(下述)。

**电流环(最内,最高带宽)**:`id_ref=0`(磁场定向,不浪费励磁电流),`iq_ref` 由外环给定,PI 输出 dq 电压。单环验证时,堵转固定角度下 id/iq 为干净直流(id≈0、iq 精确跟踪指令),证明 Clark/Park 与电流环数学正确。

但电流环只控转矩、不控转速——这是一个必须理解的边界:空载时恒转矩会使电机持续加速,反电动势随转速线性升高(`E = Ke·ω`),不断吃掉母线电压裕度;当电流环输出电压饱和(顶到 `Vdc/√3`)后失去控制能力,电流畸变 + PI 积分饱和,最终撞电源限流(CC)。这正暴露了串级控制的必要性:转速这个状态量缺少反馈约束,需要外环补上。

**速度环 / 位置环(外环)**:速度环用编码器测速做反馈、输出 `iq_ref`;位置环用多圈累加角做反馈、给定目标。

### 工程决策:位置环 PD 直控扭矩,绕过速度环

标准三环把主控交给速度环,但实测发现速度环在低速是致命弱点:速度由编码器角度微分得到,低速时单位时间转过的角度极小,量化噪声占比爆炸;为压噪声被迫重滤波,而重滤波引入相位滞后、吃掉相位裕度,导致闭环自激振荡。试过降增益、加大测速窗口、上 PLL 观测器,均治标不治本(完整排查见 [DEBUGGING.md](./DEBUGGING.md))。

最终的解法是做减法:

> 位置 θ 是编码器直接测的、干净的;速度 ω 是微分出来的、脏的。那就让干净的位置当主控,脏的速度只做辅助阻尼。

```c
// 位置环 PD/PID 直接输出电流指令,绕过速度环
iq_ref = Kp_pos * theta_err          // 位置误差(干净)→ 主力
       + pos_integral                // 积分项,克服 cogging 稳态卡死
       - Kd_pos * omega_filt;        // 速度阻尼(脏,可重滤波,仅防过冲)
```

P/I 项吃干净的位置误差(低速不抖),D 项的速度只管阻尼(滞后无所谓)。低速抖动由此根治。这也是许多关节伺服低速段采用"位置直控扭矩(PD/PID)"而非严格串级三环的原因——是踩坑后的工程判断,而非套用公式。

## 6. CAN 分布式主从协议

两块板通过 CAN 总线组成主从架构,验证人形机器人多关节分布式控制的通信链路。

**报文契约**:

| 报文 | ID | 方向 | DLC | 内容 |
|---|---|---|---|---|
| 位置指令 | 0x101 | 主 → 从 | 4 | float 目标角(多圈累加角) |
| (预留)力反馈 | 0x201 | 从 → 主 | 4 | float iq(L3 未实现) |

主板读自身编码器作为"手轮",单圈累加成多圈角后,以 1ms 周期经 CAN 下发;从板收到后喂入位置环的目标,用三环跟随,支持多圈、顺逆双向联动。

**float 装进 CAN 数据段**:CAN 数据段是 `uint8_t[8]`,float 需按字节拆装,用 union 实现:

```c
typedef union { float f; uint8_t b[4]; } float_bytes_t;
```

(两端同为 F446 小端,union 直拷最省事;跨异构 MCU 时工业界常定点化为 int32 规避字节序问题。)

**量产思路:同款固件,角色区分主从**。主从板烧录同一套 FOC 固件,仅靠角色配置(发送 / 接收)区分——主板只发不收,从板只收不动。这呼应工业伺服模组"一套固件管所有关节、靠节点 ID 区分"的设计。

### CAN 关键配置

**位时序 / 波特率**(两板必须一致):

```
PCLK1 = 45MHz,目标 500kbps
Prescaler=6,BS1=12Tq,BS2=2Tq,SJW=1
验证:45M / (6 × (1+12+2)) = 500kbps ✓
采样点:(1+12) / (1+12+2) = 86.7%(落在推荐的 75~87.5% 区间)
```

采样点故意靠后,是为了等信号在总线上传播稳定后再采,容错更好(对比 UART 固定 50% 中点采样)。

**标准帧 ID 过滤**:bxCAN 滤波寄存器按 29 位扩展 ID 格式对齐,11 位标准 ID 需 `<< 5` 推到寄存器高位 `bit[31:21]`,从板用掩码模式只放行 0x101。

**物理层**:总线两端各一个 120Ω 终端电阻做阻抗匹配(只放一端会反射,并成 60Ω 负载过重发不出显性位);双板共地、双绞线。
