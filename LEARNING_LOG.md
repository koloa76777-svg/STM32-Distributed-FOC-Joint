# 学习历程

> 返回 [README](./README.md)

本项目从零手写,不依赖现成 FOC 库,历经多个迭代节点逐步搭建。以下是开发时间线与每个阶段的关键收获,完整的每日技术笔记维护于个人学习库。

---

## 时间线总览

| 阶段 | 节点 | 关键产出 |
|---|---|---|
| W1 | 环境搭建 | STM32CubeIDE + Nucleo-F446 环境,裸 HAL 三路 PWM + AS5047P SPI 读取调通 |
| W2 | 中断框架 | TIM3 25kHz Update 中断驱动 SPWM,消除主循环时间抖动 |
| W2 | 开环 FOC | 手写逆 Park + SVPWM 注入法 + 编码器零位校准,真实电角度接入,开环力矩控制跑通 |
| W3 | 电流环 | 三相电流采样(TRGO 同步 + DMA)→ Clark/Park → 电流环 PI,从电压开环进化到电流闭环 |
| W3 | 角度同步 | θe 中断同步架构(SPI+DMA 非阻塞 + 错误自愈),根治快慢环不同步 |

## 各阶段技术要点

### W1-W2 · 底层框架与开环 FOC

- 搭建 TIM2/TIM3 主从同步 PWM(中心对齐,25kHz),TIM3 Update 中断作为 FOC 的固定节拍。
- 把 FOC 运算从主循环搬进 25kHz 中断,理解了"为什么 FOC 算法要放在 PWM 中断里":节拍一致性(编码器/电流/PWM 同步)、避免主循环与中断的资源竞争、延迟可控(实时性要求确定性)。
- 吃透 SVPWM 几何原理:8 个开关状态、6 个非零矢量、内切圆半径 `Vdc/√3`、母线利用率比 SPWM 高 15.5%;并理解了 dq 坐标系的本质——把 αβ 系里高速交变的正弦量,变成 dq 系里两个静止的直流量,从而能用 PI 无静差控制。
- 机械角 vs 电角度:电角度 = 机械角 × 极对数(7),并用实测数据验证(机械角动 13° → 电角度动约 94°)。

### W3 · 电流环

- 谷底同步采样:中心对齐 PWM 谷底时三相下管全开,电感电流经续流流过分流电阻,远离开关沿,是采相电流的最佳时刻。
- 用 `ia + ib + ic ≈ 0`(基尔霍夫电流定律)作为电流采样正确性的黄金自检。
- 理解电流环为何用 PI 不用 PID:D 项对采样噪声敏感,而电流采样本身带开关噪声;以及抗积分饱和(windup)的必要性。
- 理解三环带宽分离:电流环最内、带宽最高(接近 PWM 频率),速度环、位置环逐层降频,内环带宽需远高于外环 5~10 倍。

### W3 · 角度同步与三环

- 发现"加不加 printf 电机行为就变",定位到 θe 异步更新与控制中断不同步,改用 SPI+DMA 在中断内同步读取根治。
- 实测 HAL SPI+DMA 在高频中断下约 80μs 的最小可靠重启间隔,据此对编码器读取分频,并加入错误自愈机制。
- 低速测速量化误差:M 法测速在低速时量化噪声占比爆炸,这是物理极限,滤波只能平滑表象、无法增加信息量;改用 M 法多点测速(环形缓冲)从源头降噪。
- 闭环滤波失稳:滤波引入相位滞后,在闭环里吃掉相位裕度,过度滤波反而自激振荡——开环与闭环对滤波的结论相反。
- 最终用位置环 PD 直控扭矩取代严格串级三环,根治低速抖动。

### W4 · CAN 分布式

- 吃透 CAN 三大考点:非破坏性逐位仲裁(显性覆盖隐性、ID 小者胜出且报文无损)、位时序与采样点计算(86.7% 采样点)、滤波器掩码/列表模式。
- 报文契约设计:主→从位置指令帧(0x101),float 经 union 拆字节装入 CAN 数据段。
- 通信层与控制层解耦:从板拿到的就是一个目标角度数,等价于本地调试改目标——CAN 只换了目标的来源,W3 的三环位置环一行不动。

## 这个项目教给我的

几条贯穿始终、影响了我工程方法的认知:

- **现场观察优先于理论推断**:多次用直接测量(万用表实测引脚电压、断电手转电机判断编码器方向、堵转观察 id/iq)推翻了基于理论的错误猜测。硬件现实优先于纸面分析。
- **严肃逐行对比代码**:多个耗时数小时的 bug,最终都靠与"上一个能跑的版本"逐行对比定位,而非凭直觉猜原因。
- **减法 > 加法**:卡死和振荡的解法,常常是删掉一个引入噪声/延迟的环节(如砍掉低速噪声大的速度环),而非不断叠加补偿。
- **源头降噪 > 末端滤波**:同一个滤波器,信号脏时怎么调都振荡,信号干净时轻处理就稳——优先改善信号源,而非在末端硬滤。
- **工业级容错思维**:读取链路出错要能自愈(检测锁死 → Abort → 复位状态机 → 下一拍恢复),而非死等或整机复位。
